# 学科知识整合智能体：基于知识图谱、选择式压缩与混合检索的多教材整合实验报告

## 摘要

多教材整合的核心矛盾在于：教师需要压缩阅读和备课成本，但教学系统不能牺牲概念覆盖、来源可追溯性与问答可信度。本文提出一个面向 7 本教材整合任务的智能体系统：先解析教材并构建知识图谱，再通过 embedding 召回、LLM 精判和 Union-Find 聚类完成跨教材知识点整合，最后使用“原文精选”而非“全文改写”的方式生成不超过 30% 的精华版本。RAG 问答始终检索原始教材 chunk，并使用向量召回、BM25 与 Reciprocal Rank Fusion 融合排序。当前可复现实验显示：7 本教材全量流程下，系统将 832 个原始知识点整合为 288 个知识点，压缩比 5.84%；RAG 自建 10 题测试集上 keyword_hit@5 与 book_hit@5 均为 100%。这些结果说明，选择式压缩和原文检索分离能够在黑客松约束下同时满足压缩、可追溯和问答可靠性。

## 1. 问题分析

赛题要求把 7 本教材整合为不超过原始体量 30% 的精华版本，并提供知识图谱、RAG 问答和 Agent 架构说明。该任务与普通摘要不同，存在四个挑战：

1. 知识点颗粒度难统一。章节标题过粗，句子级片段过细，需要抽取可教学、可连接的原子概念。
2. 跨教材重复知识点难判定。同名概念可能含义不同，异名概念可能指向同一知识。
3. 压缩不能破坏引用。如果用 LLM 重写正文，虽然可读性强，但引用真实性下降。
4. RAG 需要高召回和严格来源。教材问答不能用模型常识替代教材证据。

因此，本系统将“整合产物”和“原文检索库”解耦：整合产物服务教师阅读和图谱审阅，RAG 始终服务于原文证据。

## 2. 方案设计

### 2.1 Hybrid Function-Agent 架构

系统将任务划分为确定性函数和语义决策 Agent：

- 函数模块：解析、切块、embedding、检索、RRF 融合。
- LLM 函数：知识点抽取、RAG 回答。
- Agent 模块：跨教材整合 Agent、教师对话 Agent。

该设计避免为了形式化 Agent 而引入重型框架，同时保留了可解释的工具调用边界。

### 2.2 知识图谱构建

节点为 2-12 字原子概念，边包括 `prerequisite`、`contains`、`parallel`、`applies_to` 四类。抽取 prompt 强制输出 JSON，后处理阶段丢弃非法边，并绑定教材、章节和来源片段。

### 2.3 跨教材整合

整合算法包括：

1. 使用 BGE-small-zh 对节点定义和上下文编码。
2. 对不同教材节点做 top-k 相似候选召回。
3. LLM 按名称、定义、上下文、关系四要素判断同一概念。
4. Union-Find 合并同义节点。
5. 在 30% 预算内选择原文片段，生成 merge/keep/remove 决策。

### 2.4 RAG Pipeline

RAG 使用章节感知 chunk，目标长度 500-800 字，overlap 50-100 字。检索阶段同时计算向量相似度和 BM25 分数，再用 RRF 融合。回答 prompt 明确要求只基于上下文回答，每个事实附教材和章节引用，证据不足时拒答。

### 2.5 Obsidian 式双链

系统在整合报告中使用 `[[知识点]]` 作为双链表达，前端节点详情可显示教材来源、章节与相关节点。该设计把知识图谱与精华教材连接起来，使教师可以从图谱跳到原文证据。

## 3. 实验与结果

### 3.1 实验环境

| 项目 | 配置 |
| --- | --- |
| 后端 | FastAPI + SQLite |
| 前端 | React + ECharts |
| LLM | ModelScope API Inference，支持外部 provider fallback |
| Embedding | BGE-small-zh 本地推理 |
| 部署 | Docker / ModelScope Studio |

### 3.2 跨教材整合实验

| 指标 | 结果 |
| --- | ---: |
| 教材数 | 7 |
| 原始节点 | 832 |
| 原始边 | 737 |
| 整合后节点 | 288 |
| merge 决策 | 20 |
| keep 决策 | 268 |
| 原始字符数 | 3,267,041 |
| 保留字符数 | 190,930 |
| 压缩比 | 5.84% |

结果说明：7 本教材全量流程已经满足 30% 压缩硬约束。当前预算仍较宽，因此没有触发强制 remove；若扩展到更多教材或更严格预算，remove 决策会被启用。

### 3.3 RAG 检索测试集

自建测试集包含 10 个教材相关问题。评测脚本记录 top-5 chunk 是否命中问题关键词与预期教材。

| 指标 | 结果 |
| --- | ---: |
| chunks | 6113 |
| embedded chunks | 6113 |
| eval cases | 10 |
| keyword_hit@5 | 100% |
| book_hit@5 | 100% |
| elapsed | 60.1s |

结果说明：混合检索能同时捕获语义问题和教材术语。由于评测集规模较小，本结果主要作为黑客松可复现实验，不等价于大规模教育问答基准。

### 3.4 压缩策略成本对比

| 策略 | 估算 token | 引用真实性 | 风险 |
| --- | ---: | --- | --- |
| LLM 全文改写 | 2.8M | 中 | 幻觉、引用失真、成本高 |
| 原文精选 + 元数据 | 0.56M | 高 | 可读性依赖片段选择 |

原文精选方案约节省 5 倍 token 成本，并使每个精华片段都能回到教材原文。

### 3.5 工程鲁棒性实验

系统实现了 ModelScope 多模型 fallback：当主模型遇到 quota、429、unavailable 等错误时，可尝试 `模型ID:外部提供方` 形式的外部 provider，例如 DeepSeek 或 XiaomiMiMo。RAG 回答在 LLM 不可用时退化为带引用的抽取式答案，从而避免演示现场空响应。

## 4. 局限性与未来方向

1. 自建 RAG 测试集只有 10 题，后续应扩展到事实题、比较题、机制推理题和跨教材题。
2. 部分教材在 API 不可用时使用启发式图谱兜底，后续应对低置信章节自动重抽。
3. SQLite 向量存储适合单机小规模，后续大规模教材库可迁移到 Chroma、FAISS 或 Milvus。
4. 当前 merge 判定依赖 embedding 阈值和可选 LLM，后续可以加入人工标注小样本，计算 precision、recall 和 Cohen's kappa。
5. 当前图谱主要用于可视化与整合解释，未来可探索 GraphRAG，把路径约束纳入检索和回答。
