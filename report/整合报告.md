## 2. 整合方法

系统采用两阶段整合：

1. 召回阶段：使用 BGE-small-zh 向量对不同教材中的知识点做相似候选召回。
2. 判定阶段：LLM 根据名称、定义、上下文、关系四要素判断是否为同一概念。
3. 聚类阶段：通过 Union-Find 合并高置信重复概念。
4. 压缩阶段：保留教材原文片段，不重写正文；只生成 merge/keep/remove 决策、理由和来源元数据。

## 3. 决策摘要

| 决策类型 | 含义 | 当前数量 |
| --- | --- | ---: |
| merge | 多教材重复或等价概念合并为一个全局知识点 | 20 |
| keep | 独特或教学必要概念保留 | 268 |
| remove | 低重要度且超出预算的概念暂不纳入 | 0 |

当前压缩比低于 30%，因此未触发强制 remove；系统仍保留 remove 能力，用于更大规模教材或更严格预算场景。

## 4. 典型 merge 决策案例

### 案例 1：[[炎症]]

- 问题：不同教材对炎症的定义可能分别从病理变化、免疫反应、局部组织反应角度表述。
- 决策：merge。
- 理由：名称一致，定义核心均指向损伤因子引发的局部防御反应，相关节点包括充血、渗出、白细胞反应。
- 教学价值：合并后可形成跨教材统一入口，同时保留各教材原文差异。

### 案例 2：[[发热]]

- 问题：教材可能从体温调节、生理反馈或疾病表现角度解释发热。
- 决策：merge。
- 理由：上下文均位于体温调节和病理生理反应，关系上共同连接炎症介质、下丘脑和体温升高。
- 教学价值：适合在图谱中作为机制型节点展示前因后果。

### 案例 3：[[内环境]]

- 问题：不同教材对内环境可能使用“细胞外液”“稳态”等近邻概念。
- 决策：根据定义和上下文做 merge 或 keep。
- 理由：只有当定义明确指向细胞直接生活的液体环境时合并；若教材讨论稳态调节机制，则作为相关但不等同节点保留。
- 教学价值：避免“同名合并”和“近义误合并”的常见错误。

## 5. 图谱统计与来源保留

整合过程不会删除原始教材库。系统同时保存：

1. `corpus_raw`：原始章节 chunk，用于 RAG 引用。
2. 局部图谱：单本教材节点和边，用于来源追踪。
3. 全局图谱：跨教材聚类后的整合视图。
4. 决策表：记录 action、confidence、reason 和 affected_nodes。

这种分离保证了压缩视图不会破坏原文检索和来源证据。

## 6. 教学完整性说明

当前策略不是按字符粗暴截断，而是以知识点重要度和跨教材覆盖为核心：

- 多本教材共同出现的核心概念优先保留。
- 与多个边相连的高中心性节点优先保留。
- 教材独有但定义完整、对章节理解必要的概念保留。
- 预算不足时优先移除低重要度、低连接度、来源片段过长的节点。

## 8. 可复现口径

1. 前端上传 7 本教材或把教材放入 `data/textbooks/` 后调用解析流程。
2. 为每本教材运行 `/api/graph/build/{textbook_id}`。
3. 调用 `/api/integrate/run` 生成整合结果。
4. 通过 `/api/integrate/status`、`/api/integrate/decisions` 和 `/api/integrate/summary` 复核统计。
